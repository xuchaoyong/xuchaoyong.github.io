1. How is the deconstruct function is called?

Every function would have a stack when they are called by another function, the input argument and calling function return address would be stored in the stack of called function. When the called function has executed all its statement, it will return to the calling function. At this time, all the local or automatic variables, such as build-in type or object, would be freed from the called function stack when the destructure function would be called. There are some scopes when the deconstruct function would be called:

  • block
  • function scope

2. Unwinding stack method of try-catch exception

throw statement processes objects put on the stack by the entire sequence of function calls between the try block and the throw.
Throw mechanism is that, just as with function returns, the class destructors are called for any automatic class objects on the stack.

If the throw statement return from lower level stack to a higher level stack, it should free all the local variables in the intermediate function stack.

Hexo Deploy without Password

Posted on 2015-03-24   |  

1. Background

The SSH is the default way to accompish remote login in linux system, which use RSA/DSA encryption to guarantee the system safety. There is a series of tools, such as ssh-keygen, to generate private and public key. By coping the public key to the remote machine, you will get much convenience without inputing the password everytime when you try to connect to the remote server.

2. Solutions

1. Step one: Modify the hexo global _config.yml file.
2. Step two: generate key

ssh-keygen -t rsa -C "your_email@example.com"

When you get the info:

Keep it empty so that you would not need to enter password when you execute hexo deploy.

3. Step three: start ssh-agent and add key
4. Step four: copy the ~/.ssh/id_rsa.pub content to your github ssh key.
5. Done.

3. Conclusion

By generate ssh private and public key, there is no need for you to enter password when you want to hexo deploy operation.
Enjoy it!
